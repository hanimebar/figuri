# Figuri — An Äctvli Product

> Plain-English monthly financial narratives for small-practice accountants. In their client's language.

## What It Does

Figuri turns a set of monthly financial figures into a clear, friendly, 200–260 word narrative that an accountant sends to their small business client. The narrative is generated by Claude in the client's chosen language (English, French, German, Finnish, Spanish at launch), edited inline by the accountant in under 60 seconds, and dispatched via Resend as a branded email — with the accountant's booking link in the footer.

The problem it solves: 72% of small business clients want their accountant to explain their financial position proactively (Xero 2025). Only 28% receive it. The monthly management accounts PDF lands in the client's inbox unread. Figuri makes the explanation automatic, professional, and multilingual.

**Domain target:** figuri.io (verify availability) or figuri.actvli.com

## Product Scope — MVP Only

Four features. Nothing else.

1. **Client roster** — accountant stores each client: name, email, business type, standing context notes, language setting, optional booking link URL
2. **Monthly figure entry form** — 8 fields per client, ~90 seconds to complete
3. **Narrative generation** — Claude Haiku generates 200–260 word plain-English narrative in the client's language; accountant edits inline
4. **Send** — Resend dispatches branded HTML email to the client with booking link in footer

**Explicitly excluded from MVP:** Xero/QBO OAuth integration, open rate tracking, client-facing portal or login, team accounts, historical charts, PDF export, mobile app, multi-currency, custom templates, automated data import.

## Data Model

```
accountants
  id                          uuid PK
  email                       text UNIQUE
  business_name               text
  logo_url                    text (Supabase Storage, optional)
  reply_to_email              text (verified with Resend)
  default_booking_link_url    text (optional)
  stripe_customer_id          text
  stripe_subscription_status  text (trialing | active | past_due | canceled)
  trial_ends_at               timestamptz
  created_at                  timestamptz DEFAULT now()

clients
  id               uuid PK
  accountant_id    uuid FK → accountants
  name             text
  email            text
  business_type    text (e.g. "retail florist", "sole trader plumber")
  context_notes    text (seasonality, goals, watch points — standing context)
  language         text DEFAULT 'en'  -- ISO 639-1 code
  booking_link_url text (optional, overrides accountant default)
  created_at       timestamptz DEFAULT now()

monthly_figures
  id                  uuid PK
  client_id           uuid FK → clients
  accountant_id       uuid FK → accountants
  period_month        text (e.g. "2026-03")
  revenue             numeric
  cost_of_goods       numeric (nullable — service businesses leave blank)
  gross_margin_pct    numeric (nullable)
  operating_expenses  numeric
  net_profit          numeric
  cash_position       numeric
  one_off_items       text (optional free text)
  context_note        text (optional — accountant's extra context for this month)
  created_at          timestamptz DEFAULT now()

narratives
  id                      uuid PK
  figures_id              uuid FK → monthly_figures
  client_id               uuid FK → clients
  accountant_id           uuid FK → accountants
  language                text  -- copied from client at generation time
  body_text               text  -- the generated + edited narrative
  status                  text (draft | sent)
  generated_at            timestamptz DEFAULT now()
  sent_at                 timestamptz (nullable)
  booking_link_included   boolean DEFAULT false
```

RLS: all tables gate on `accountant_id = auth.uid()`. No client-facing auth in MVP.

## Multi-Language Architecture

### Launch languages (5)
`en` English, `fr` French, `de` German, `fi` Finnish, `es` Spanish

### Post-launch additions (weeks 2–4)
`nl` Dutch, `it` Italian, `pt-BR` Brazilian Portuguese, `sv` Swedish, `nb` Norwegian

### How it works
- Language is a field on the `clients` record, selected once by the accountant
- Claude generates the narrative **natively** in the target language — not translated from English
- Historical narratives store their `language` value immutably — changing a client's language only affects future narratives
- Spanish gets a region sub-selector: `es-ES` (Spain) vs `es-419` (LATAM) — 30 min of work, prevents terminology variance

### Pre-launch requirement
Generate 20 test narratives before launch (4 per language, different business types). Review with a native speaker in each language. Do not skip this.

## Claude Prompt Design

### System prompt
```
You are a plain-English financial narrative writer for accountants.
Your job is to write a short monthly update that an accountant sends
to their small business client. The update summarises the client's
financial performance for the month in clear, friendly, non-jargon
language that a business owner — not an accountant — can understand
and act on.

Rules you must follow without exception:
1. Write in [LANGUAGE]. Use vocabulary and number formatting natural
   to a native [LANGUAGE] speaker in a professional but warm context.
2. Write exactly 200–260 words. No more, no less.
3. Describe what the numbers show. Do not speculate about why the
   numbers are what they are unless the accountant has explicitly
   stated the reason in the context notes.
4. Do not use accounting jargon. Never use: EBITDA, amortisation,
   accruals, prepayments, deferred income, or similar technical terms.
5. Do not make predictions about future performance unless the
   accountant has provided forward-looking context.
6. Do not offer tax advice, legal advice, or any opinion that
   implies professional liability.
7. End every narrative with one open question specific to the numbers
   provided — not a generic question.
8. Format numbers according to [LANGUAGE] locale conventions.
   German/French/Finnish/Spanish: period as thousand separator,
   comma as decimal (e.g. 1.234,56).
   English: comma as thousand, period as decimal (e.g. 1,234.56).
9. Do not repeat the phrase "this month" more than twice.
10. Tone: warm and professional — like a trusted adviser, not a machine.
```

### User message (constructed per-generation)
```
Write a monthly financial update for the following client.

Client name: [client_name]
Business type: [business_type]
Standing context: [context_notes — or "None provided"]
Month being reported: [e.g. "March 2026"]

Financial figures:
- Revenue: [revenue]
- Cost of goods sold: [cogs or "Not applicable — service business"]
- Gross margin: [gross_margin_pct% or "Not applicable"]
- Operating expenses: [operating_expenses]
- Net profit: [net_profit]
- Cash in bank at month end: [cash_position]
- One-off items: [one_off_items or "None"]
- Accountant's additional context: [context_note or "None"]

Currency: [GBP / EUR / USD — from accountant settings]

Write the narrative now, following all rules in your instructions.
```

## Booking Link — Technical Implementation

Zero engineering. The accountant pastes their own Calendly/Tidycal/Acuity URL into:
- `accountants.default_booking_link_url` (practice-wide default)
- `clients.booking_link_url` (per-client override, optional)

On send, the API route checks: client link → practice default → omit if neither.

Email footer: *"Have questions about these figures? [Book a 20-minute review call →]"*

Figuri never owns scheduling.

## Known Risks

| Risk | Mitigation |
|---|---|
| Claude mis-states a figure in the narrative | Show entered figures alongside the editable narrative; ToS makes accountant responsible for review before send |
| Resend deliverability from shared domain | Require accountant email/domain verification during onboarding; provide step-by-step DKIM/SPF instructions |
| Low trial-to-paid conversion | Monthly cron on 5th: "8 clients haven't received a Figuri update this month →" keeps the habit alive |
| Spanish terminology variance (Spain vs LATAM) | `es-ES` / `es-419` sub-selector on language field |
| Fathom builds a lite tier at sub-£30/month | Move fast to establish community presence in AAT/ICB before Fathom notices the sub-niche |
